Overview

Sora is an automated trading signal generator running as a single asyncio Python process. It combines a deterministic technical analysis engine with a multi-agent LLM debate system to produce high-quality trading signals for US equities (via Alpaca) and cryptocurrencies (via Binance/CoinGecko).

Core principle: trade less, trade higher quality. Every signal passes through multiple quality gates — technical indicators, market regime detection, confidence scoring, confluence validation, and a hard 8-rule gate — before delivery via Telegram.

Key architectural decisions:

  • Single-process asyncio — no external broker or task queue
  • SQLite for persistence — no database server required
  • All confidence scores deterministic and explainable
  • LLM is augmentation layer, not core decision-maker
  • Resilient to data source failures (graceful fallbacks throughout)

Features

Multi-Market Coverage

  • US equities: NASDAQ/NYSE symbols via Alpaca Markets API (daily OHLCV bars)
  • Cryptocurrency: BTC, ETH, SOL, BNB, AVAX, LINK, DOT, MATIC, ADA, XRP via Binance public API (no key required)
  • Market regime detection using SPY (equities) or BTC (crypto) as proxies

Deterministic Confidence Scoring

Seven explainable dimensions weighted to produce a 0–100 score:

  • Trend strength (ADX, EMA alignment)
  • Signal alignment (RSI, MACD, Bollinger Bands)
  • Volatility quality (ATR % of price)
  • Volume confirmation (ratio, trend)
  • Market regime fit
  • Historical performance (win rate, Sharpe)
  • Drawdown state

Market Regime Detection

Classifies market as BULL, BEAR, NEUTRAL, RANGING, or VOLATILE using ADX, trend slope, and volatility vs historical baseline. Each regime influences eligible strategies and confidence calculation.

Confluence Validation

Before accepting any signal, validates that technical factors agree: ADX strength, EMA alignment, RSI consistency, volume confirmation, and regime compatibility. Alignment ratio must be ≥50%.

Signal Gate (8 Hard Rules)

Pre-notification filter all BUY/SELL signals must pass:

  • Mandatory: R:R ≥ 1.5:1, entry zone active
  • Scorecard: confidence floor, RSI max, volume confirm, catalyst cooldown, SL sanity
  • Allows 1 failure out of 5 scorecard checks

Multi-Agent LLM Debate

  • Bull Agent: builds strongest bullish case
  • Bear Agent: stress-tests bull thesis, finds hidden risks
  • Analyst Agent: synthesizes debate → final verdict
  • Debate scoring: Bear rates bull thesis 0–100

Moomoo Framework

5-step structured analysis: Objective → Data Collection → Multi-Dimensional Analysis → Synthesis → Structured Report. Includes valuation, entry strategy, and risk management.

Pre-Execution Filters

  • Earnings calendar proximity (blackout within 3 days)
  • Liquidity check (min 300K avg vol, penny stock filter)
  • Volume environment (dead market, spike detection)
  • Bid-ask spread check
  • ATR pre-gate (min 0.3% daily range)
  • Earnings proximity guard (HIGH risk flag ≤5 days)

Automated Scanning

  • US equities: pre-market (8:30 ET) + pre-close (15:00 ET)
  • Crypto: every 4 hours
  • Position monitoring: 30 min (US), 2 hours (crypto)
  • Weekly review: autonomous strategy evaluation

Position Tracking

  • Log entries with price, SL, TP, qty
  • Automatic SL/TP monitoring and alerts
  • Close positions on SL hit or TP reached

LLM-Powered Memory

  • Karpathy-style wiki: persistent LLM-maintained files
  • Wiki ingestion: notes and feedback merged automatically
  • Wiki linting: automated consistency checking
  • Autoresearch loop: weekly review of signal accuracy

User Profile

Auto-extracted from wiki: risk tolerance, preferred timeframes, favorite strategies, max position size, common mistakes.

Feedback Loop

  • Trade outcomes logged (took/skip/partial + reasons)
  • Signal outcomes tracked (3d, 7d, 14d returns)
  • Outcome-adjusted conviction fed into LLM context
  • Lessons extracted from user behavior patterns

Telegram Chat Interface

  • Free-form trading chat with LLM-powered responses
  • Tool-using agent: analyze, signals, watchlist, lessons
  • Chat history maintained for conversational context

Rich Signal Reports

  • HTML-formatted messages with entry zones, exit targets, R:R
  • Confidence breakdown visualization
  • Technical condition assessment (RSI, Stoch, Williams %R)
  • Full LLM reports with bull case, bear case, catalysts

Chart Generation

  • Candlestick charts via matplotlib/mplfinance
  • EMA21 and EMA55 overlays
  • Entry zone highlight (green band)

  • Dark theme (dark background, green/red candles)

Strategy Engine

Strategy engine uses conditional scoring. Each strategy defines boolean checks, counts how many pass, and computes (conditions_met / total_checks) * 100. A minimum threshold must be met for signal emission.

StrategyCategoryDifficultyCompatible RegimesMin ConditionsRisk (SL/TP)
EMA + RSI + VolumeTrendEasyBULL, NEUTRAL4/66%/12%
Supertrend + MACDTrendMediumBULL, VOLATILE, NEUTRAL3/55%/10%
Bollinger SqueezeBreakoutMediumBULL, VOLATILE, RANGING3/57%/14%
RSI Mean ReversionMean Rev.EasyRANGING, VOLATILE, NEUTRAL2/44%/7%

StrategySelector iterates all regime-compatible strategies, evaluates each, selects highest blended score (signal confidence weighted against optional historical confidence).

Memory System

Implements a Karpathy-style LLM wiki: persistent markdown files read-and-updated by the LLM itself each cycle.

knowledge/ wiki/ strategy.md # Active trading strategies and rules patterns.md # Chart patterns the LLM has identified lessons.md # Accumulated lessons from review cycles regime.md # Market regime observations symbols/ AAPL.md # Per-symbol wiki pages BTC.md raw/ inputs.log # Raw input log for audit trail

Wiki Ingestion

When user submits a note or trade feedback, LLM reads current wiki + new input, decides which sections to update, returns JSON mapping filenames to full new content. Only changed files written.

Wiki Linting

LLM-driven audit checks for contradictions, stale claims, orphan pages, data gaps. Returns structured issue report with severity levels.

Weekly Review

ReviewAgent evaluates all signals from past week against actual outcomes, identifies which strategies over/underperformed, updates confidence weights, extracts new behavioral patterns. Results merged into lessons.md.

Installation

Clone, install, configure, run in minutes:

# Clone the repository
git clone <repository-url>
cd noor-telegram-bot

# Create virtual environment
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.template .env
# Edit .env with your API keys

# Initialize the database
python -c "from db.store import init_db; init_db()"

# Start the bot
python bot.py

Extending

Adding a New Strategy

  1. Create a new class in engine/strategies.py inheriting from Strategy
  2. Define name, description, category, difficulty, compatible_regimes, risk_params, min_confidence
  3. Implement evaluate() returning a StrategySignal or None
  4. Register it: add register(YourStrategy()) at module level

Adding a New Data Source

  1. Create a module in data/ returning bars: {"time": int, "open": float, "high": float, "low": float, "close": float, "volume": float}
  2. Update _fetch_bars() in orchestrator.py to dispatch to new source
  3. Update _detect_market() and crypto symbol set

Adding a New Analysis Module

  1. Create a module in analysis/
  2. Import and call from _run_full_analysis() in orchestrator.py
  3. Data is passed to Bull/Bear/Analyst agents as context
